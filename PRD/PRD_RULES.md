# 📜 กฎการทำงาน PRD (Product Requirements Document)

> **เอกสารนี้ต้องถูกอ่านและปฏิบัติตามทุกครั้งก่อนเริ่มทำงานเกี่ยวกับ PRD**

---

## 1. หลักการห้ามคาดเดา (Zero Assumption Policy)

- ❌ **ห้ามคาดเดาข้อมูล** — หากไม่มีข้อมูลเพียงพอ ต้องถามกลับเสมอ
- ❌ **ห้ามเติมข้อมูลเอง** — ข้อมูลทุกอย่างต้องมาจากผู้ใช้หรือแหล่งอ้างอิงที่ได้รับอนุญาตเท่านั้น
- ✅ **ถามกลับให้ได้มากที่สุด** — หากข้อมูลไม่ชัดเจน ให้รวบรวมคำถามและถามกลับทั้งหมด

---

## 2. แหล่งอ้างอิงที่อนุญาต (Approved Reference Sources)

แหล่งอ้างอิงที่อนุญาต **ต้องมาจากโฟลเดอร์ `ref/` เท่านั้น** ประกอบด้วย:

| ไฟล์                      | คำอธิบาย                                    |
| :------------------------ | :------------------------------------------ |
| `project_summary_v3.md`   | สรุปภาพรวมโครงการ SiamGroup V3 (Core & HRM) |
| `BACKEND_ARCHITECTURE.md` | สถาปัตยกรรม Backend (Pure PHP, Model-Based) |
| `FRONTEND_STANDARDS.md`   | มาตรฐานการพัฒนา Frontend (React, MUI, Vite) |
| `THEME_SUMMARY.md`        | สรุป Theme และ Design Tokens                |
| `Meeting HR 23 Feb.txt`   | สรุปการประชุม HR วันที่ 23 ก.พ.             |
| `siamgroup_v3_final.sql`  | โครงสร้างฐานข้อมูล V3 (SQL Schema)          |

### กฎการใช้แหล่งอ้างอิง:

1. หากต้องการข้อมูลจากแหล่งอื่นนอกเหนือจาก `ref/` → **ต้องถามผู้ใช้ก่อนเท่านั้น**
2. หากเอกสารอ้างอิงหลายไฟล์มีข้อมูลเรื่องเดียวกันแต่เนื้อหาต่างกัน → **ต้องถามผู้ใช้ก่อนว่าจะใช้ข้อมูลจากไฟล์ไหน**
3. หากข้อมูลไม่พอ → สามารถค้นหาข้อมูลเพิ่มเติมได้ แต่ **ต้องถามก่อนว่าใช้ข้อมูลที่ค้นหามาเป็นแหล่งอ้างอิงได้หรือไม่**

---

## 3. การเสนอความคิดเห็น (Opinion Proposal)

- ❌ **ห้ามเสนอความคิดเห็นโดยไม่ถาม** — ทุกข้อเสนอต้องถามก่อนว่า "โอเคไหม" เสมอ
- ✅ **ระบุเหตุผลประกอบ** — เมื่อเสนอสิ่งใดต้องอธิบายว่าทำไม พร้อมให้ผู้ใช้ตัดสินใจ

---

## 4. รูปแบบเอกสาร (Document Format)

- ไฟล์ PRD ต้องเป็นรูปแบบ **Markdown (.md)**
- เอกสารทั้งหมดต้องเขียนเป็น **ภาษาไทย**
- ไฟล์เก็บไว้ที่ **`PRD/`** เท่านั้น

---

## 5. ขั้นตอนการทำงาน (Workflow)

```
1. อ่านกฎเอกสารนี้ (PRD_RULES.md) ทุกครั้งก่อนเริ่มงาน
2. อ่านและทำความเข้าใจเอกสารอ้างอิงใน ref/
3. ระบุข้อมูลที่ขาดหาย → รวบรวมคำถาม → ถามผู้ใช้
4. รอคำตอบจากผู้ใช้ → ตรวจสอบความสอดคล้องกับเอกสารอ้างอิง
5. เขียน PRD ตามข้อมูลที่ได้รับ
6. ส่งให้ผู้ใช้ตรวจสอบ → แก้ไขตาม Feedback
```

---

## 6. การจดบันทึกการคุย (Conversation Notes)

> **บังคับ:** ทุกครั้งที่มีการคุยเกี่ยวกับ PRD ต้องบันทึกสิ่งสำคัญไว้เสมอ

- ✅ **จดบันทึกการคุยที่จำเป็น** ไว้ที่ `../PRD_conversation_notes.md`
- ✅ **อัปเดตทุกครั้ง** ที่มี PRD สมบูรณ์ หรือมีการตัดสินใจสำคัญ
- ✅ **วัตถุประสงค์:** เพื่อให้กรณีขึ้นแชทใหม่สามารถอ่านและต่องานได้ทันที

### ข้อมูลที่ต้องบันทึก:

1. **สถานะ PRD แต่ละตัว** — ตัวไหนเสร็จแล้ว / กำลังทำ / ยังไม่เริ่ม
2. **สรุปสิ่งที่ตกลงแล้ว** — เฉพาะประเด็นสำคัญ (ไม่ต้องคัดลอก PRD ทั้งหมด)
3. **คำถามที่ค้างอยู่** — ถ้ามี Open Questions ให้จดไว้
4. **กฎสำคัญ** — สรุป PRD_RULES.md แบบย่อ
5. **Conversation IDs** — เก็บ ID ของแชทที่เกี่ยวข้องไว้สำหรับอ้างอิง
