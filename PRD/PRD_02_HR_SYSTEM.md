## 1. ภาพรวม (Overview)

ระบบ HR Management ให้ฝ่าย HR, หัวหน้างาน และผู้บริหาร สามารถ:

- จัดการข้อมูลพนักงาน
- ดูรายงานเวลาทำงาน
- ตั้งค่าตารางกะ / วันหยุด / สิทธิ์ลา
- อนุมัติคำร้องต่างๆ
- ดูรายงานสรุป

### 1.1 ผู้ใช้งาน

| กลุ่มผู้ใช้         | การเข้าถึง                              |
| :------------------ | :-------------------------------------- |
| HR                  | เข้าถึงทุกเมนู HR                       |
| IT                  | เข้าถึงทุกเมนู HR (`is_cross_company`)  |
| หัวหน้า / ผู้บริหาร | เข้าถึงตาม `core_level_permissions`     |
| พนักงานทั่วไป       | ❌ ไม่มีสิทธิ์เข้าเมนู HR (ใช้ PRD #01) |

> **อ้างอิง:** สิทธิ์ทั้งหมดควบคุมโดย `core_level_permissions` → ดู PRD #00

## 4. จัดการตารางกะ (Work Schedule Management)

### 4.1 ที่เก็บข้อมูล

> **ตาราง:** `hrm_work_schedules`

### 4.2 ประเภทกะ

| ประเภท     | `schedule_type` | คำอธิบาย                           |
| :--------- | :-------------- | :--------------------------------- |
| กะตายตัว   | `FIXED`         | เวลาเข้า-ออกคงที่ เช่น 08:00-17:00 |
| กะยืดหยุ่น | `FLEXIBLE`      | นับชั่วโมงทำงาน เช่น 8 ชม./วัน     |

### 4.3 Pattern สัปดาห์

| Pattern    | `week_pattern` | คำอธิบาย                      |
| :--------- | :------------- | :---------------------------- |
| ทุกสัปดาห์ | `ALL`          | ทำงาน/หยุดเหมือนกันทุกสัปดาห์ |
| สัปดาห์คี่ | `ODD`          | เช่น หยุดเสาร์สัปดาห์คี่      |
| สัปดาห์คู่ | `EVEN`         | เช่น หยุดเสาร์สัปดาห์คู่      |

> **ใช้สำหรับ:** เสาร์เว้นเสาร์ → `day_of_week = 6` + `week_pattern = ODD/EVEN`

### 4.4 กะข้ามวัน

- `is_cross_day = 1` → เลิกงานข้ามวัน (เช่น เข้า 21:00 ออก 06:00)
- ระบบจะคำนวณ `work_date + 1` สำหรับเวลาออก

### 4.5 ฟังก์ชัน HR

- ตั้งกะให้พนักงานแต่ละคน (เลือก 7 วัน)
- กำหนด `effective_from` / `effective_to`
- Copy กะจากพนักงานคนอื่น
- ตั้งกะแบบ Bulk (เลือกหลายคนพร้อมกัน)

### 4.6 ตัวอย่างข้อมูล

| 10          | 3 (พ.)      | ALL          | 0          | 08:00      | 17:00    |

### 6.1 ประเภทลา

> **ตาราง:** `hrm_leave_types`

**ฟังก์ชัน:**

- เพิ่ม / แก้ไข / ลบ ประเภทลา
- กำหนดว่าต้องแนบไฟล์หรือไม่ (`requires_file`)

> **หมายเหตุ `is_paid`:** ย้ายไปกำหนดที่ **ตอนส่งคำขอลา** (`hrm_leave_requests.is_paid`) แทน — ไม่ hardcode ที่ประเภทลา เพื่อให้ทุกประเภทลาเลือกได้ว่าจะรับค่าจ้างหรือไม่

**ตัวอย่างข้อมูล `hrm_leave_types`:**

| name_th             |  requires_file  | หมายเหตุ                           |

## 11. OT — ข้อมูลที่ทราบแล้ว (บันทึกไว้สำหรับ PRD #03)

> **หมายเหตุ:** ส่วนนี้บันทึกข้อมูลจาก Meeting HR เรื่อง OT — จะ Finalize ใน PRD #03

### 11.1 OT ออฟฟิศ

- ต้อง **เกิน 1 ชั่วโมงเต็ม** จึงจะได้ OT
- เงินเดือน < 20,000 บาท: ชั่วโมงละ **70 บาท** (หรือคำนวณ เงินเดือน ÷ 30 ÷ 8)
- เงินเดือน ≥ 20,000 บาท: ชั่วโมงละ **90 บาท**
- OT **1 เท่า** (วันธรรมดา) / OT **1.5 เท่า** (วันหยุดนักขัตฤกษ์)

### 11.2 OT เซลล์

- ชั่วโมงละ **90 บาท** (พี่แนนกำหนด)

### 11.3 OT Operation (CNX) — ทำเหมือนกันก่อน

> **SAR vs SDR → ใช้กฎเดียวกันก่อน → ค่อยแก้ทีหลังเมื่อสรุปกับพี่แนนเรียบร้อย**

**ข้อมูลที่ทราบ (CNX):**

| ช่วงเวลา    | เพิ่ม (บาท) |
| :---------- | :---------- |
| 17:01-19:59 | +100        |
| 20:00-22:00 | +200        |
| 22:01-23:59 | +300        |
| 00:00-01:59 | +500        |
| 02:00-03:59 | +600        |
| 04:00-05:00 | +300        |
| 05:01-06:00 | +200        |
| 06:59-07:59 | +100        |

### 11.4 ทำงานวันหยุด

- วันหยุดปกติ: ได้ **1 เท่า** เพิ่ม (เงินเดือน ÷ 30)
- วันหยุดนักขัตฤกษ์: ได้ **1.5 เท่า** เพิ่ม

### 11.5 สลับวันหยุด

## 12. ระบบประเมินผลงาน / โบนัส (Performance Evaluation)

> **อ้างอิง Meeting HR:** "มีการแจ้งเตือนระบบการประเมิณผ่านงานในแต่ละเดือน / โบนัส แจ้งเตือนผ่านหน้าระบบและเมลล์"

### 12.1 ภาพรวม

- หัวหน้างานประเมินผลงานลูกน้อง **ทุกเดือน**
- แจ้งเตือนผ่าน: **หน้าระบบ** + **Email**
- HR ดูผลประเมินรวมทุกคน + Export ได้

### 12.2 ขั้นตอน

1. **ต้นเดือน** → ระบบส่งแจ้งเตือนให้หัวหน้างาน (ตาม `manager_id`)
2. **หัวหน้างาน** → เข้ามากรอกประเมินผลงานลูกน้องแต่ละคน
3. **HR** → ดูผลประเมินรวม, สรุปรายงาน, Export
4. **ปลายปี** → HR ดึงคะแนนเฉลี่ยทั้งปี → ใช้ประกอบการพิจารณาโบนัส

### 12.9 กฎบริษัท — การมาสาย/ขาดงาน

| กฎ                                  | รายละเอียด                           | ผลในระบบ                                  |
| :---------------------------------- | :----------------------------------- | :---------------------------------------- |
| ห้ามสายเกิน **15 นาที** ต่อครั้ง    | สายเกิน 15 นาที = ถือว่าผิดกฎ        | ระบบแจ้งเตือน HR + บันทึก                 |
| ห้ามสายรวมเกิน **30 นาที** ต่อเดือน | นาทีสายรวมทั้งเดือนต้องไม่เกิน 30    | ระบบแจ้งเตือน HR เมื่อใกล้ถึง/เกิน        |
| **ห้ามขาดงาน** เด็ดขาด              | บริษัทมีวันลาเยอะ — ถ้าจะไม่มาต้องลา | ขาดงาน = **บังคับหักเป็นลา** (HR ตั้งค่า) |

> **Logic:** ถ้าไม่มา + ไม่มีใบลา → ระบบตั้งสถานะ **Absent** → HR สามารถ **บังคับเปลี่ยนเป็นลา** (ลากิจ/ลาอื่นๆ `is_paid = 0`) ได้

### 12.10 โบนัส — แนวทางการคำนวณ

> **โบนัสคำนวณจาก 2 ส่วน:**

| ส่วน                            | แหล่งข้อมูล                                      | คำอธิบาย                                  |
| :------------------------------ | :----------------------------------------------- | :---------------------------------------- |
| **1. คะแนนประเมินเฉลี่ยทั้งปี** | `hrm_evaluations.weighted_score` เฉลี่ย 12 เดือน | คะแนนจากหัวหน้า (5 หมวด × ถ่วงน้ำหนัก)    |
| **2. คะแนนจากขาด/ลา/มาสาย**     | `hrm_time_logs` + `hrm_leave_requests`           | ยิ่งมาตรงเวลา ไม่ขาด ไม่ลา → คะแนนยิ่งสูง |

**ตัวอย่างสัดส่วน (เสนอ):**

| องค์ประกอบ                       | สัดส่วน  | คะแนนเต็ม |
| :------------------------------- | :------: | :-------: |
| คะแนนประเมินผลงาน (เฉลี่ยทั้งปี) |   70%    |    70     |
| คะแนน Attendance (ขาด/ลา/สาย)    |   30%    |    30     |
| **รวม**                          | **100%** |  **100**  |

**เกณฑ์คะแนน Attendance (เสนอ — สอดคล้องกับกฎบริษัท):**

| เกณฑ์     | คะแนนเต็ม | วิธีคิด                                                                    |
| :-------- | :-------: | :------------------------------------------------------------------------- |
| ไม่ขาดงาน |    10     | ขาด = **หัก 5 คะแนน/ครั้ง** (กฎห้ามขาดเด็ดขาด)                             |
| ไม่ลาบ่อย |    10     | หัก 0.5 คะแนน/วันลา (เกินโควตาเกณฑ์)                                       |
| ไม่มาสาย  |    10     | หัก 1 คะแนน ถ้าสายเกิน 15 นาที/ครั้ง, หัก 2 คะแนน ถ้ารวมเกิน 30 นาที/เดือน |

> ⏳ **สัดส่วนและเกณฑ์จริง → Finalize ใน PRD #03 (ระบบเงินเดือน)** — เนื่องจากการจ่ายโบนัสเป็นส่วนของ Payroll
